MVC Separando funciones 1 MVC Introducción El patrón de arquitectura MVC, se refiere a una manera de organizar el código.

La principal característica es separar el almacenamiento de los datos de la capa de visualización.

Podéis ver la lista de frameworks que trabajan sobre este patrón Utiliza tres componentes: 1.​ Modelo: encargado del manejo de los datos 2.​ Vista: encargada de la parte visual e interactiva con el usuario 3.​ Controlador: la lógica de la aplicación y enlace entre el Modelo y la Vista La mejor manera de entender como funciona es con un ejemplo.

Este patrón de arquitectura utiliza otro patrón de diseño, el Observer, que es el necesario para notificar los cambios entre las tres partes.
